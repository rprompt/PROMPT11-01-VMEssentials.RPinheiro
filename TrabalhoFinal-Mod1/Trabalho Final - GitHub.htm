Trabalho Final

Objectivos

O objectivo do trabalho final é a realização dum servidor HTTP (Hypertext Transfer Protocol) para visualização e navegação dos artefactos contidos em conjuntos de assemblies, nomeadamente namespaces, tipos e membros.

Requisitos

Contextos

O servidor apresenta informação sobre conjuntos de assemblies, designados por contextos. Cada contexto é definido por uma pasta, em que:

  • O nome do contexto corresponde ao nome da pasta.
  • Os assemblies constituintes do contexto correspondem aos ficheiros presentes nessa pasta.

URIs

O servidor deve aceitar pedidos de método GET aos URI caracterizados em seguida:

  • / - raiz.

    • O resultado do GET é uma lista com ligações para todos os contextos existentes.

  • /{ctx} - contexto de nome ctx.

    • O resultado do GET inclui
      • Ligação para a lista de assemblies.
      • Ligação para a lista de namespaces.

  • /{ctx}/as - assemblies do contexto ctx.

    • O resultado do GET é uma lista com ligações para os assemblies deste contexto.

  • /{ctx}/ns - namespaces do contexto ctx.

    • O resultado do GET é uma lista com ligações para os namespaces deste contexto.

  • /{ctx}/as/{assemblyName} - assembly definido num contexto.

    • O resultado do GET inclui
      • Informação sobre o assembly (friendly name, public key).
      • Lista com ligações para os tipos definidos neste assembly , organizada por namespace.

  • /{ctx}/ns/{namespacePrefix} - namespace definido num contexto.

    • O resultado do GET é uma lista com ligações para sub-namespaces e tipos pertencentes a este namespace.

  • /{ctx}/ns/{namespace}/{shortName} - tipo.

    • O resultado do GET inclui:
      • Características do assembly (e.g. nome, nome completo);
      • Ligação para o assembly onde está definido;
      • Ligação para o namespace onde pertence;
      • Listas com os membros do tipo, agrupadas por tipo de membro (e.g. método, campo, propriedade).
    • O shortName é o nome CTS do tipo, e não o nome usado no C#.

  • /{ctx}/ns/{namespace}/{shortName}/m/{methodName} - lista de métodos com o mesmo nome.

    • Resultado do GET é uma lista de métodos (sobrecargas) com o nome methodName.
    • Para cada sobrecarga, a representação retornada deve incluir a informação do método presente na metadata, nomeadamente a informação sobre os parâmetros.

  • /{ctx}/ns/{namespace}/{shortName}/c - lista de construtores.

    • Resultado do GET é uma lista dos construtores do tipo.
    • Para cada construtor, a representação retornada deve incluir a informação presente na metadata, nomeadamente a informação sobre os parâmetros.

  • /{ctx}/ns/{namespace}/{shortName}/f/{fieldName} - campo.

    • Resultado do GET é uma representação com a informação do campo presente na metadata, nomeadamente o nome e o tipo do campo.

  • /{ctx}/ns/{namespace}/{shortName}/p/{propName} - propriedade.

    • Resultado do GET é uma representação com a informação da propriedade presente na metadata, nomeadamente o nome, o tipo e os parâmetros da propriedade.

  • /{ctx}/ns/{namespace}/{shortName}/e/{eventName} - evento.

    • Resultado do GET é uma representação com a informação do evento presente na metadata, nomeadamente o nome e o tipo do evento.

As representações HTML devem conter ligações para os URI associados a cada artefacto apresentado. Por exemplo, a inclusão do nome dum tipo deve estar associada a uma ligação para o URI com a descrição desse tipo.

Arquitectura da solução

Sugerem-se os seguintes aspectos de desenho da solução:

  • Utilização do HttpListener para a recepção dos pedidos HTTP e a transmissão das respostas.

  • Realização dum componente de encaminhamento com as seguintes características

    • Associação de templates (e.g. /{ctx}/ns/{namespace}/{shortName}/e/{eventName}) a handlers responsáveis por processar os pedidos cujo URI safisfaça os templates.
    • Extracção e conversão das variáveis dos templates (e.g. ctx, namespace, shortName e eventName), e passagem desta informação aos handlers (e.g. através de parâmetros de métodos).

  • Realização dum componente para a definição declarativa de documentos HTML.
